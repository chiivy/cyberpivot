# Contributing to CyberPivot

CyberPivot is open source and community contributions are welcome. This document explains how to contribute effectively.

Content contribution guidelines are coming soon. For now contributions are limited to code, bug reports, and feature requests.

---

## Code contributions

Bug fixes, new features, accessibility improvements, performance. Check issues tagged good-first-issue for a starting point. Read [CLAUDE.md](./CLAUDE.md) for stack and conventions before you start.

---

## Salary data and market information

If you have verified current data for a specific market or role, open an issue tagged data-update with your source.

---

## Community resources

Podcast recommendations, blogs, YouTube channels, communities worth following. Open an issue tagged resources.

---

## How to Submit

1. Fork the repository
2. Create a branch: `git checkout -b fix/your-change-name`
3. Make your changes following the conventions in [CLAUDE.md](./CLAUDE.md)
4. Submit a pull request with a clear description of what you changed and how you tested it

Pull requests that do not follow project conventions will be asked to revise before merging.

---

## Code of Conduct

Be direct. Be honest. Be respectful. This platform exists to help people build careers in cybersecurity regardless of their background or location. Contributions that serve that mission are welcome. Contributions that undermine it are not.

---

## Questions

Open an issue on GitHub or start a discussion in the Discussions tab.
